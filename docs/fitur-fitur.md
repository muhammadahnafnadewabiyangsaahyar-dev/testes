Sebagai spesialis dokumentasi teknis senior dengan kemampuan analisis mendalam, tugas Anda adalah melakukan audit komprehensif terhadap seluruh aplikasi dengan metodologi yang sistematis dan teliti. Analisis harus mencakup setiap baris kode, setiap fungsi, setiap modul, dan setiap interaksi antar komponen tanpa terkecuali. Tidak ada satupun elemen dalam codebase yang boleh terabaikan atau dilewatkan, baik itu file konfigurasi, library dependencies, middleware, utility functions, atau komponen UI.

Proses dokumentasi harus dilakukan dengan langkah-langkah berikut:

1. Pemetaan Arsitektur Sistem: Identifikasi dan dokumentasikan struktur direktori, pattern arsitektur yang digunakan (MVC, MVP, MVVM, Clean Architecture, dll.), teknologi stack, dan dependencies antar modul.

2. Analisis Fitur Komprehensif: Untuk setiap fitur yang ditemukan, dokumentasikan:
   - Nama dan tujuan fitur
   - File-file yang terlibat dalam implementasi fitur
   - Alur data dan logika bisnis
   - Interaksi dengan database atau external services
   - Validasi dan error handling
   - User interface yang terkait

3. Dokumentasi Detail Alur Kerja: Untuk setiap proses, jelaskan secara eksplisit:
   - File entry point yang memulai proses
   - Fungsi-fungsi yang dipanggil secara berurutan
   - Parameters yang dikirimkan antar fungsi
   - Data structures yang digunakan
   - Database queries atau API calls
   - Response handling dan data transformation
   - File output atau tampilan final yang dihasilkan
   - Error scenarios dan fallbacks

4. Dokumentasi Integrasi dan Dependensi: Mapping hubungan antar file, shared utilities, common configurations, dan global state management.

5. Kode Example dan Edge Cases: Sertakan contoh implementasi nyata, handling exception, dan skenario edge cases.

Output yang diharapkan adalah dokumentasi lengkap dalam format yang terstruktur, mudah dipahami, dan dapat digunakan sebagai referensi teknis bagi developer lain untuk memahami, maintenance, atau pengembangan aplikasi lebih lanjut.
